Three open-source effects · Buildless · Zero-dependency

Liquid Glass

Three different ways to make the web look like glass — one with pure CSS & SVG, two with WebGL. Each is its own drop-in project.

Start with the CSS/SVG refraction engine, add the WebGL2 droplet that floats over your content, or fill the whole surface with the WebGL2 foam field. Pick one, or use all three.

The collection

Each effect is a self-contained module with its own playground and GitHub repo. Here’s what makes each one different.

CSS + SVG · no canvas

Liquid Glass

The original. Real backdrop-filter refraction driven by a per-element SVG displacement map — edge bending, chromatic aberration, frost and a specular rim on any DOM element. No canvas, no WebGL, no build step. The refraction needs a Chromium browser; Safari/Firefox fall back to a clean frost.

How it works Playground GitHub
backdrop-filter refraction
WebGL2 · canvas

Liquid Bubble

A discrete 3D glass droplet that floats above a grid of images and magnifies and refracts what’s beneath it — a metaball lens with chromatic aberration, a Fresnel rim and a moving specular hotspot. It free-drifts and springs to your cursor. Runs everywhere WebGL2 does, including mobile.

Playground GitHub
WebGL2 · canvas

Liquid Foam

Where the bubble is one droplet, the foam fills the whole surface with drifting, merging glass bubbles — a tinted sheet that refracts a living backdrop, with a clear pocket your cursor carries through it. Fully tunable: coverage, bubble size, drift, tint. A great ambient hero background.

Playground GitHub

CSS vs WebGL — which to reach for

The big split: the first effect paints with the browser’s own filter pipeline; the other two run a shader on a <canvas>.
